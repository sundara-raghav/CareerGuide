# Revenue Model & Placement Pitch — CareerGuide India

## Revenue Model

### Phase 1: MVP (0–6 months)
- **Free for all students** — build user base, gather real feedback
- Target: 1 district pilot with 500 students

### Phase 2: Pilot Revenue (6–18 months)

| Tier | Customer | Price | Model |
|---|---|---|---|
| B2G District License | State Education Dept. | ₹5–15 lakh/district/year | SaaS subscription |
| NGO/School Bundle | 100-school pack | ₹50K–2L/year | Per-institution |
| Premium Counseling | Individual families | ₹499–999/session | Marketplace |

### Phase 3: Scale (18+ months)

| Revenue Stream | Description | Potential |
|---|---|---|
| **B2G SaaS** | State-level rollout for government schools | ₹1–5Cr per state/year |
| **B2B White-label** | NGOs, edtech companies license the engine | ₹10–50L/partner |
| **Analytics Dashboard** | College enrollment trends sold to institutions | ₹2–5L/college/year |
| **Scholarship API** | Scholarship providers pay for targeted matching | Revenue share |
| **Employer Integration** | Companies post apprenticeships to engaged students | ₹1–5L/company/campaign |

### Unit Economics (steady state)
- CAC (via B2G govt contracts): ~₹0 (tendered)
- LTV per district: ₹5L+ annually
- Infrastructure cost per 10K students: ~₹15K/month

---

## B2G Strategy — How to Win Government Contracts

1. **Pilot with proof**: Offer 1 free district, measure enrollment lift
2. **Align with NEP 2020**: Frame as "career guidance as mandated by National Education Policy"
3. **Government reporting**: Build district-level dashboards ready for DC / DM presentations
4. **CMGGA / DPIIT grants**: Apply for Startup India + state education tech grants
5. **Impact metrics**: Track and publish → students enrolled in higher edu / scholarships won / first-gen graduates

## Placement Pitch — Modules That Showcase Skills

### For Backend Engineering Roles
- Blueprint-based modular Flask architecture
- Service/repository separation (clean architecture)
- SQLAlchemy ORM with complex queries
- Supabase integration + RLS policy design
- Rate limiting, CSRF, input sanitization

### For ML Engineering Roles
- Ensemble pipeline: RF + XGBoost + LR + rules
- Stacking with calibrated confidence
- SHAP explainability
- Feature engineering (marks, aptitude, demographics)
- Synthetic dataset generation + train/eval pipeline
- Top-K ranking with business rule post-processing

### For DevOps / Platform Engineering Roles
- Multi-stage Dockerfile
- Docker Compose (web + celery + redis + nginx)
- GitHub Actions CI/CD (5-stage pipeline)
- Nginx reverse proxy with security headers
- Gunicorn production config
- Health checks + container orchestration

### For Data Engineering Roles
- 1200-record synthetic dataset generation
- Feature engineering pipeline (sklearn transformers)
- Evaluation: accuracy, F1 macro, top-K accuracy
- Fairness considerations (income-stratified analysis)
- Model versioning + artifact management

### For Product / UX Roles
- 4-step onboarding wizard (validated UX pattern)
- 56-question quiz engine (6 sections)
- Google Maps college explorer
- Career roadmap visualization
- Parent explainability dashboard
- Dark/light mode, GSAP animations, mobile-first

### For Public Impact / Social Tech Roles
- Targets rural/government school students
- Multilingual architecture
- Offline-ready PWA hooks
- B2G deployment model for government scale
- District-level analytics for policy decisions
